Summary of Migration and Key Changes

Data Types:
- Oracle NUMBER(10) → PostgreSQL BIGINT
- Oracle VARCHAR2 → PostgreSQL VARCHAR
- Oracle DATE DEFAULT SYSDATE → PostgreSQL TIMESTAMP DEFAULT CURRENT_TIMESTAMP
- Oracle NUMBER(10,2) → PostgreSQL NUMERIC(10,2)
- Oracle NUMBER(5) → PostgreSQL INTEGER

Auto-Increment:
- Used GENERATED ALWAYS AS IDENTITY for EMP_ID instead of sequence and trigger.

Procedures/Functions:
- Converted Oracle procedure to PostgreSQL function (CREATE OR REPLACE FUNCTION).
- Transaction control (COMMIT, ROLLBACK) is managed outside functions in PostgreSQL.

Indexes:
- Syntax for index creation is compatible.

Privileges:
- Syntax for granting privileges is compatible.

Performance Optimization:
- Use EXPLAIN ANALYZE in PostgreSQL to compare and optimize query performance.

Additional Considerations:
- Map Oracle data types to PostgreSQL equivalents.
- Familiarize with PostgreSQL functions and operators.
- Understand string handling differences: Oracle's UPPER() = PostgreSQL's UPPER().
- Date and time handling differences: Oracle's NOW() = PostgreSQL's CURRENT_TIMESTAMP().
- Subqueries syntax differences: Oracle's IN clause = PostgreSQL's array containment operator.
- Joins syntax differences: Oracle's JOIN = PostgreSQL's JOIN keyword.
- Indexes comparison: Check available indexes and usage in PostgreSQL.
